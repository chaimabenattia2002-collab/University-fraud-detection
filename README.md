University Fraud Detection System
A real-time fraud detection system for universities using Apache Kafka and Apache Spark (PySpark). It simulates student activity events and automatically detects suspicious behavior as it happens.

🔍 What It Detects
Multiple Location Logins - Same student logging in from 3+ different locations/IPs
Exam Cheating - Multiple students accessing an exam from the same IP address
Account Sharing - Same student logging in from 3+ different devices
🎲 Fake Data Generation
The system uses the Faker library to simulate realistic university data including:

50 simulated students (STU1000 - STU1049)
5 courses (CS101, CS102, MATH201, PHYS101, ENG101)
3 exams (CS101 Midterm, CS102 Final, MATH201 Midterm)
Random IP addresses (192.168.1.x range)
6 device types (Windows PC, MacBook, iPhone, Android, iPad, Linux PC)
Random city locations generated by Faker
200 total events sent per run with a 30% fraud rate
📊 Event Types
login - Student login with IP, device and location
exam_access - Student accessing an exam
attendance - Student attending a course
📁 Project Structure
kafka_producer.py - Simulates and sends student events to Kafka
spark_consumer_clean.py - Reads events in real-time and flags fraud using Spark
requirements.txt - All Python dependencies
setup.sh - Checks and installs everything needed
start_kafka_system.sh - Launches all components automatically
⚙️ Requirements
Python 3.8+
Java 11
Apache Kafka
Apache Spark
🚀 How To Run
Install dependencies: pip install -r requirements.txt

Run setup: bash setup.sh

Start the full system: bash start_kafka_system.sh

🛠️ Built With
Apache Kafka - Event streaming pipeline
Apache Spark (PySpark) - Real-time stream processing
Faker - Realistic fake student data generation
kafka-python - Kafka producer integration
